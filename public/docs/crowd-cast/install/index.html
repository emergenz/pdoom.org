Install crowd-cast

crowd-cast ships as a single binary with screen-capture (libobs) embedded, so you don’t need to install OBS or anything else separately. Pick your platform below.

Download for macOS Download for Windows Download for Linux

You can always grab the latest builds from the GitHub releases page.

  • macOS
  • Windows
  • Linux

  1. Open CrowdCast.dmg and drag the app to your Applications folder.

  2. Launch CrowdCast from Applications. The setup wizard opens automatically on first run.

  3. Grant the two macOS permissions the wizard asks for:

    • Screen Recording: required to capture windows.
    • Accessibility: required to record keyboard and mouse input.

  4. Choose your preferences (apps to record, start-at-login). See First-run setup for the full walkthrough.

  5. Click Done. The app restarts and appears as a small icon in your menu bar, and recording starts automatically. To attribute your contributions, open the tray menu and choose Sign in with Google (the wizard does not prompt for this).

Note

On first launch, crowd-cast downloads its capture components once (this needs internet, and a notification shows while it runs). After that they’re cached, and updates install automatically in the background.

  1. Run crowd-cast-setup.exe. The installer places the app under your user profile, with no admin rights or UAC prompt required.

  2. SmartScreen warning: because the app isn’t yet widely distributed, Windows may show “Windows protected your PC”. Click More info, then Run anyway.

  3. Choose your preferences (apps to record, start-at-login). See First-run setup for details.

  4. On first launch, crowd-cast downloads its capture components automatically (one-time, needs internet). The app relaunches itself when done. This takes about 30 to 60 seconds.

    Caution

    Don’t close the app during the first-launch download. If it is interrupted, delete the install folder at %LOCALAPPDATA%\Programs\crowd-cast and re-run the installer.

  5. After the relaunch, crowd-cast appears in your system tray (bottom-right, near the clock), and recording starts automatically. To attribute your contributions, open the tray menu and choose Sign in with Google (the wizard does not prompt for this).

Linux support is limited. crowd-cast currently supports exactly two sessions: GNOME on Wayland (per-app application capture) and sway (full-screen capture only). On any other session (X11, KDE, or other wlroots compositors such as Hyprland) the agent will not start: the setup wizard blocks until you log into a supported session.

The Linux installer is a user-space install into ~/.local. It downloads the signed Linux appcast, verifies the binary and libobs bundle, adds a menu entry, and enables start-at-login. It needs sudo only once, to add you to the input group for input capture.

  1. Run the published installer:

    If the installer reports that a required tool is missing, install that package with your distro’s package manager and run the command again.

  2. Launch crowd-cast. The wizard checks the current session, screen-capture components, the input group, and (on GNOME) the focus extension. If it flags anything, follow the instructions it shows.

    • If the wizard tells you to log out and back in, do that once, then launch crowd-cast again.
    • On GNOME, installing the bundled focus extension from the wizard and activating the input group both take effect on that one relogin. After that, your selected apps capture with no picker.
    • On sway, crowd-cast records full-screen (the desktop portal asks which output to share each time it starts), and the tray needs an SNI host, so run waybar (swaybar shows the icon but won’t open its menu).

Developer build

From a checkout, use packaging/linux/install.sh --local to install your local build outputs instead of the published release.
